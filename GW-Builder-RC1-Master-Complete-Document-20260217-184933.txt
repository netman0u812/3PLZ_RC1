1. Executive Summary
============================================================

GW-Builder is a multi-tenant secure gateway platform for Rocky Linux 9 that provides:

- Per-tenant VRF isolation
- Route-based IPsec (IKEv2 PSK) site-to-site connectivity
- Per-tenant BGP routing (FRR)
- Namespace-preserving DNS rewrite plane
- Forward proxy (Squid) for generic egress
- Reverse proxy enforcement plane with per-user mTLS
- Per-tenant PKI (Root CA → Tenant Intermediate → User/Server certs)
- nftables firewall enforcement model
- Optional inspection export and SSH proxy add-ons
- Unified per-tenant service catalog registry

The system is designed to operate without external IAM/SAML dependencies in MVG.

============================================================
2. Design Goals
============================================================

Security Goals:
- Complete routing isolation between tenants (no route bleed)
- Strong authentication via mTLS per user (SAN=email identity)
- Explicit application exposure (no implicit forwarding)
- Decryptability for inspection and policy enforcement
- Revocation enforcement (CRL-based)

Operational Goals:
- Deterministic tenant lifecycle (create, validate, export, delete)
- Canonical configuration source of truth
- Idempotent teardown
- Operator-driven onboarding workflow
- Minimal external dependencies

Architectural Goals:
- Modular add-on capability
- Separation of runtime vs documentation vs planning artifacts
- Clean promotion path: RC → GA

============================================================
3. High-Level Architecture Overview
============================================================

Tenant Network
    ↓
IPsec IKEv2 PSK (VTI)
    ↓
Per-Tenant VRF
    ↓
BGP (FRR)
    ↓
DNS Rewrite Plane (Unbound)
    ↓
Reverse Proxy (mTLS enforcement)
    ↓
Forward Proxy (Squid) for generic egress
    ↓
Upstream Services / Internet

============================================================
4. Core Architectural Planes
============================================================

4.1 Transport Plane
- strongSwan IKEv2 PSK
- Route-based VTI interfaces
- /30 tunnel addressing
- /32 loopback termination model
- Vendor compatibility hooks (Cisco IOS / Arista EOS)

4.2 Routing Plane
- Per-tenant VRF
- FRR (zebra + bgpd)
- eBGP adjacency per tenant
- Explicit route boundaries

4.3 Namespace Plane
- Tenant DNS VIP
- Conditional forwarding
- Canonical rewrite file:
  /etc/gateway/catalog/<tenant>/dns-rewrites.conf

4.4 Enforcement Plane
- Reverse proxy per app
- mTLS user authentication
- Per-app server certificates
- CRL enforcement

4.5 Egress Plane
- Squid explicit forward proxy
- CONNECT support
- Optional transparent mode

4.6 Firewall Plane
- nftables baseline
- Per-tenant policy hook chain
- Allow-list guardrails (inspection + SSH add-on)

4.7 Inspection Plane (Optional Add-on)
- Export modes: off / log / tap
- Bidirectional mirroring
- Guardrail enforcement

============================================================

    ↓

- CRL enforcement by proxy layer

GW-Builder RC1 Architecture Addendum: Tenant Onboarding
======================================================

This addendum extends the RC1 architecture documentation to include the full
customer onboarding workflow required to provision a tenant.

1. Intake and Data Collection
-----------------------------
All tenant provisioning begins with completion of the Customer Onboarding Worksheet:

- IPsec peer vendor + public IP
- IKE loopback identity
- PSK exchange
- /30 tunnel addressing
- Customer ASN + prefixes
- DNS conditional forwarding domains + resolvers
- Requested proxy services (forward + reverse + SSH)
- User list (SAN=email) for mTLS enrollment

2. Translation into Platform Objects
------------------------------------
Worksheet fields map directly into platform registry and configuration:

- Tenant VRF + addressing → gwctl add-tenant
- IPsec peer block → strongSwan tenant include
- BGP neighbor → FRR VRF stanza
- DNS overrides → /etc/gateway/catalog/<tenant>/dns-rewrites.conf
- Reverse proxy apps → reverse-proxy.d/*.app entries
- SSH proxy targets → ssh.d/ service entries (add-on)
- User SAN identities → PKI issuance workflow

3. Canonical Service Catalog
----------------------------
All tenant service intent is stored under:

/etc/gateway/catalog/<tenant>/

This is the single source of truth for:
- App proxies
- DNS rewrites
- Inspection export mode
- SSH proxy allow-lists

4. Operational Validation
-------------------------
After onboarding, operators execute:

- gwctl tenant-health <tenant>
- RC1 execution worksheet test cases
- Export + backup validation

5. Security Outcomes
--------------------
- Namespace preservation: tenant users use identical hostnames as inside
- mTLS enforcement: per-user authentication at proxy boundary
- VRF isolation: no route bleeding across tenants
- Optional inspection export: decrypted traffic mirroring under guardrails

GW-Builder Bug Triage & Severity Rubric
======================================

Severity Levels:
S0 Blocker - Prevents install or tenant bring-up; no workaround.
S1 Critical - Breaks isolation/security enforcement; workaround difficult.
S2 Major - Core function degraded (IPsec flaps, proxy failures); workaround exists.
S3 Minor - Non-critical feature bug; cosmetic or rare edge case.
S4 Enhancement - Improvement request; not a defect.

Priority Guidance:
P0 Fix immediately before GA.
P1 Fix in RC cycle.
P2 Fix post-GA patch.
P3 Backlog/roadmap.

Required Bug Fields:
- Component (IPsec/BGP/DNS/Proxy/PKI/Firewall/Add-on)
- Tenant impacted
- Repro steps
- Expected vs actual
- Logs and evidence
- Security impact assessment

11. Glossary of Key Terms
============================================================
VRF: Virtual Routing and Forwarding instance providing route isolation.
VTI: Virtual Tunnel Interface for route-based IPsec.
mTLS: Mutual TLS authentication requiring both client and server certificates.
PKI: Public Key Infrastructure (Root CA → Intermediate CA → End-Entity certs).
CRL: Certificate Revocation List used to invalidate issued certificates.
BGP: Border Gateway Protocol used for tenant route advertisement.
Squid: Forward proxy supporting HTTP/HTTPS CONNECT.
Reverse Proxy: Per-app enforcement proxy (NGINX/Envoy) terminating TLS.
Inspection Export: Optional decrypted traffic mirroring subsystem.
Service Catalog: Canonical tenant registry under /etc/gateway/catalog/<tenant>/.
MVG: Minimum Viable Gateway release baseline.
RC1: Release Candidate 1 build pending validation before GA.
GA: General Availability release.
